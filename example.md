## 整体架构

核心任务其实是两件事：**信息提取**（从字幕识别情绪点）+ **可视化输出**。

---

## 工作流拆解

**Step 1：字幕预处理**

把 SRT 文件切成"集"——按时间戳分段，每集约 300-500 行字幕合并成一段纯文本，去掉序号和时间码，只保留对话内容。这一步纯代码搞定，不需要模型。

**Step 2：情绪点提取（核心步骤）**

把每集文本喂给模型，让它输出结构化 JSON，大概长这样：

```json
{
  "episode": 1,
  "laugh_count": 2,
  "laugh_types": ["穿越失忆", "反差·无赖形象"],
  "laugh_pct": 20,
  "sweet_pct": 25,
  "hype_pct": 5,
  "conflict_pct": 10,
  "cry_pct": 30,
  "notes": "泪点来自乱世流民惨状"
}
```

**Step 3：可视化**

拿 JSON 数据渲染图表，用任何前端框架都行。

---

## 关键问题：Qwen2B 能做吗？

直说吧：**做不好**，至少在情绪分析这个任务上。

原因在于这个任务需要模型做的事情并不简单——它不只是"这句话是开心还是悲伤"的情感分类，而是需要理解剧情上下文，判断一段对话是否构成"笑点"，是什么类型的笑点（信息差、反差、小丑式角色...），并估算它在整集中的时长占比。这要求模型对叙事结构有理解能力。

2B 的小模型在这类需要推理和上下文理解的任务上，输出会很不稳定，JSON 格式也容易崩。

---

## 我的模型推荐

| 场景 | 推荐模型 | 说明 |
|------|----------|------|
| 预算有限、批量处理 | Qwen2.5-7B 或 14B（本地部署） | 7B 勉强够用，14B 稳定很多 |
| 追求质量、API 调用 | GPT-4o mini 或 Claude Haiku | 性价比最高，JSON 输出稳定 |
| 最高精度 | Claude Sonnet / GPT-4o | 理解细腻，适合做标注数据 |

实际上你可以考虑**两阶段策略**：用 Haiku 或 4o-mini 做批量初稿，再人工抽查校正，成本很低。

```
SRT文件
  ↓ 代码处理

  ↓
每集文本 → [LLM + Prompt] → JSON
  ↓

  ↓

Prompt 设计是这个流程里最关键的部分，需要给模型定义清楚什么叫"笑点"、什么叫"爽点"，并让它估算时长占比而不是只打标签。这个 prompt 设计好了，7B 的模型也能用；设计差了，Sonnet 也会输出垃圾。

## System Prompt

```
你是一位专业的短剧内容分析师，擅长分析中文短剧的情绪节奏和爽点结构。

## 情绪点定义

**笑点**：让观众发笑的内容，包含以下子类型：
- 反差：人物行为/身份/语言出现意外落差
- 自嘲&讽刺：角色自我贬低或讽刺他人
- 夸张放大：情绪/动作/表述过度夸张
- 意外&失控：突发状况打破预期
- 误会&信息差：角色之间信息不对等产生的错位
- 小丑式角色：专门负责被耍/出丑的配角
- 双关谐音：文字游戏、谐音梗
- 肢体喜剧：动作行为本身构成笑点
- 群体冲突：多人混乱场面

**甜点**：让观众感到温馨、心动的内容
- 男女主之间的互动、关心、暗示
- 家人/兄弟情谊的温情时刻

**爽点**：让观众感到解气、热血、升级的内容
- 主角反杀/怼人/打脸反派
- 事业/地位逆袭、获得认可
- 战斗胜利、展现实力

**冲突&爽点**：张力强但尚未解决的对抗
- 反派威胁、逼迫
- 主角陷入危机、被围困

**泪点**：引发悲伤、同情的内容
- 苦难、死亡、离别
- 角色的委屈、绝望

- 只输出 JSON，不要任何解释文字
- 所有百分比加起来不超过100（剩余为平淡过渡段落）
- laugh_types 只列出本集实际出现的子类型，最多4个
- notes 控制在20字以内

## 工程层面的几个建议

**1. 输出解析保险措施**

模型有时会在 JSON 前后加废话，用正则提取：

```python
import re, json

def parse_output(raw: str) -> dict:
    match = re.search(r'\{.*\}', raw, re.DOTALL)
    if match:
        return json.loads(match.group())
    raise ValueError("No valid JSON found")
```

**2. 占比校验**

```python
def validate(data: dict) -> bool:
    total = sum([
        data['laugh_pct'], data['sweet_pct'],
        data['hype_pct'], data['conflict_pct'], data['cry_pct']
    ])
    return total <= 100
```

**3. 温度设置**

情绪分析任务不需要创意，把 temperature 设低：

```
temperature: 0.2 ~ 0.3
```
